# Ruvyxa 1.0.15

Version 1.0.15 expands built-in styling and static-generation workflows while preserving the minimal
starter as the default.

## Styling

- `.scss` and `.sass` imports compile automatically in native and Node-backed pipelines.
- `.module.css`, `.module.scss`, and `.module.sass` expose deterministic, project-scoped class maps.
- Sass dependencies participate in style collection, build inputs, and development HMR.
- Framework-owned TypeScript declarations cover global and module style imports.

## Static Generation

- Dynamic routes may export a direct `staticParams` array.
- Routes with one dynamic segment may use string or number shorthand values.
- `getStaticParams` receives the real route manifest and current dynamic-segment metadata.
- `{ params, cache }` enables persistent TTL caching with dependency and route-context invalidation.

The existing `getStaticParams(): Array<Record<string, string | string[]>>` form remains supported
without enabling the new cache.

## Starters

`create-ruvyxa` packages four starters:

```bash
npm create ruvyxa@latest my-app
npm create ruvyxa@latest my-blog -- --template blog
npm create ruvyxa@latest my-admin -- --template crud
npm create ruvyxa@latest my-api -- --template api-backend
```

- `minimal` remains the default and preserves the prior scaffold behavior.
- `blog` demonstrates dynamic SSG pages with direct parameters.
- `crud` demonstrates a loader, runtime cache, API route, and validated server action.
- `api-backend` demonstrates health and item REST endpoints with consistent JSON errors.
